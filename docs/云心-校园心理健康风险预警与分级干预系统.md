# 云心-校园心理健康风险预警与分级干预系统

## 一、项目定位

本系统的核心定位，**不是心理诊断系统**，而是：

**校园心理健康风险预警与早期干预系统**

本系统的目标，是通过持续性的行为与情绪数据采集，对学生进行风险预警、趋势识别、家校联动干预和专业转介，从而在问题严重化之前，帮助学校与家长及时发现、及时防范、及时处理。

本系统必须始终坚持以下四个原则：

### 1. 只做预警，不做诊断
系统不输出“抑郁”“焦虑”“心理疾病”等诊断结论。

系统输出的是：
- 风险信号
- 趋势变化
- 预警等级
- 应对建议

### 2. 以预防为核心，而不是以判定为核心
系统重点不是给孩子定性，而是尽早发现异常走向，避免错过干预窗口。

### 3. 红色之前尽量不进入正式咨询场景
在绿色、黄色阶段，以家长陪伴、运动释放、学校观察、轻度验证为主，尽量避免过早医疗化和标签化。

### 4. 红色、黑色阶段才进入专业体系
当风险进入中高等级后，再引入心理健康公司、医院或专业团队进行评估与干预。

---

## 二、系统整体架构

系统采用三阶段闭环结构：

### 第一阶段：初步信息采集与周级预警判定
通过问卷、双摄像头系统和个体档案建立，对学生进行持续采样，并以周为区间进行预警分级。

### 第二阶段：家长端与老师端联动的轻干预与状态验证
当预警出现后，通知家长和班主任，由家长端与学校端联动进行低侵入式观察和轻干预，再根据结果决定维持、降级或升级。

### 第三阶段：专业团队介入
针对进入红色、黑色预警的学生，由心理健康公司、医院或自有专业团队接入，开展专业评估和深度干预。

**整个系统逻辑为：**
> 持续采集 → 周级判断 → 趋势确认 → 家校轻干预 → 专业转介

## 三、第一阶段：初步信息采集与周级预警判定系统

### 3.1 阶段目标

第一阶段的目标是：
1. 为每个学生建立持续性档案
2. 通过多源数据识别异常趋势
3. 以一周为单位输出风险等级
4. 为第二阶段和第三阶段提供统一入口

这一阶段回答的问题不是“这个孩子有没有心理问题”，而是：
> 这个孩子最近一周是否出现了值得关注的异常趋势

### 3.2 数据来源与系统组成

第一阶段由四个部分构成：

#### 1. 问卷系统
开学阶段进行初步问卷筛查，用于建立学生初始基线。

**作用：**
- 获取主观自报信息
- 建立初始档案
- 为后续趋势判断提供参考背景

#### 2. 摄像头系统1：实时筛查系统
该系统用于实时捕捉学生异常信号，主要负责：
- 异常事件次数

**特点：**
- 实时
- 速度快
- 准确率相对低
- 适合捕捉“发生了几次异常”

#### 3. 摄像头系统2：延迟分析系统
该系统进行高频采样与后端分析，主要负责：
- 异常占比

**特点：**
- 每日采集约300-400次表情/状态样本
- 准确率高
- 存在10分钟到半天的延迟
- 适合计算“异常在整体状态中占多少”

#### 4. 个体档案建立系统
通过身份绑定机制，将：
- 问卷数据
- 系统1数据
- 系统2数据
- 后续干预记录
统一归属到同一学生个体。

### 3.3 数据分类定义

系统将每次有效采样结果统一分为三类：

| 分类 | 说明 |
|------|------|
| 正向状态（Good） | 明显正向、积极、放松、愉快等状态 |
| 中性状态（Neutral） | 无明显好坏情绪、正常、平稳、无表情、自然状态 |
| 异常状态（Abnormal） | 系统定义的不良情绪或异常情绪状态 |

本系统关注重点不是“好情绪够不够多”，而是：
> 异常状态是否频繁、是否持续、是否上升、是否形成主导

### 3.4 统计周期与基础口径

#### 周区间
系统以7天为一个完整统计周期进行一级判断。

#### 日区间
天级数据用于辅助判断：
- 连续异常天数
- 单日峰值
- 趋势变化

#### 有效采样
只有符合质量标准、可判定的样本才进入计算。无效数据、遮挡严重、识别失败数据不计入总分母。

### 3.5 核心指标定义与计算方式

#### 3.5.1 周总有效采样数（W_Total）
**定义：** 某学生在一周内所有有效采样样本的总数。

**说明：** 所有周级比例指标的分母。

**举例：** 某学生周一到周日共采集到2450次有效样本，则 W_Total = 2450。

#### 3.5.2 周异常样本数（W_Abnormal_2）
**定义：** 某学生在一周内被系统2判定为异常状态的样本总数。

**说明：** 高精度后端分析结果中异常状态的总次数。

**举例：** 一周内2450个有效样本中，430次被判定为异常，则 W_Abnormal_2 = 430。

#### 3.5.3 周正向样本数（W_Good）
**定义：** 某学生在一周内被系统2判定为正向状态的样本总数。

#### 3.5.4 周中性样本数（W_Neutral）
**定义：** 某学生在一周内被系统2判定为中性状态的样本总数。

#### 3.5.5 周状态总和关系
```
W_Total = W_Good + W_Neutral + W_Abnormal_2
```

#### 3.5.6 周异常占比（W_Ratio）
**公式：**
```
W_Ratio = W_Abnormal_2 ÷ W_Total
```
**百分比写法：**
```
W_Ratio(%) = (W_Abnormal_2 ÷ W_Total) × 100%
```

**说明：** 核心比例指标，判断异常状态在一周内的整体严重程度。

**举例：** W_Abnormal_2 = 430，W_Total = 2450，则 W_Ratio ≈ 17.55%。

#### 3.5.7 周异常事件次数（W_Event_1）
**定义：** 某学生在一周内被系统1实时捕捉到的异常事件总次数。

**说明：** 强调“异常发生了几次”，偏向频率概念。

**举例：** 实时系统一周内累计触发异常事件86次，则 W_Event_1 = 86。

#### 3.5.8 周异常事件归一化率（W_Event_Rate）
**公式：**
```
W_Event_Rate = W_Event_1 ÷ W_Total
```
**百分比写法：**
```
W_Event_Rate(%) = (W_Event_1 ÷ W_Total) × 100%
```

**说明：** 在统一周样本基数下，系统1实时异常事件触发的频率。

**举例：** W_Event_1 = 86，W_Total = 2450，则 W_Event_Rate ≈ 3.51%。

#### 3.5.9 单日总有效采样数（D_Total）
**定义：** 某学生在某一天内所有有效采样样本的总数。

**说明：** 日级判断的基础分母。

#### 3.5.10 单日异常样本数（D_Abnormal_2）
**定义：** 某学生在某一天内被系统2判定为异常状态的样本总数。

#### 3.5.11 单日异常占比（D_Ratio）
**公式：**
```
D_Ratio = D_Abnormal_2 ÷ D_Total
```
**百分比写法：**
```
D_Ratio(%) = (D_Abnormal_2 ÷ D_Total) × 100%
```

**说明：** 判断是否构成异常日、是否出现峰值日、是否形成连续异常。

**举例：** D_Total = 360，D_Abnormal_2 = 132，则 D_Ratio ≈ 36.67%。

#### 3.5.12 异常日定义
某一天满足任一条件，即定义为异常日：

1. D_Ratio ≥ 30%
2. D_Ratio ≥ B_Day_Ratio + 10个百分点

**说明：** 一天不一定必须达到绝对30%，若明显高于自身日常水平，也可算异常日。解决“不同孩子常态不同”的问题。

#### 3.5.13 连续异常天数（Consecutive_Days）
**定义：** 当前周内连续被判定为异常日的天数。

**说明：** 异常不是偶发，而是在时间上持续存在。连续性比单次异常更能反映风险趋势。

**举例：** 周二、周三、周四连续为异常日，则 Consecutive_Days = 3。

#### 3.5.14 单日异常峰值标记（Peak_Day_Flag）
某一天满足任一条件，即记为峰值日：

1. D_Ratio ≥ 40%
2. D_Ratio ≥ B_Day_Ratio + 15个百分点

**说明：** 表示某一天异常程度明显高于平时水平，或在绝对数值上已很高。峰值日可作为黄色候选的重要触发条件，识别“某天突然明显恶化”的情况。

### 3.6 个体基线机制

#### 3.6.1 周基线异常占比（B_Ratio）
**公式：**
```
B_Ratio = 过去4个有效周的W_Ratio平均值
```

**说明：** 过去一段时间的周常态异常占比平均水平。有效周不足4周则按已有效周计算。系统上线初期可设置2-4周为基线校准期。

#### 3.6.2 周基线事件率（B_Event_Rate）
**公式：**
```
B_Event_Rate = 过去4个有效周的W_Event_Rate平均值
```

**说明：** 过去一段时间内系统1异常事件频率的平均水平。

#### 3.6.3 日基线异常占比（B_Day_Ratio）
**公式：**
```
B_Day_Ratio = 过去N天的D_Ratio平均值（建议N=14或21）
```

**说明：** 过去一段时间的日常异常占比平均水平。用于判断异常日、峰值日、某一天是否明显偏离个体常态。

#### 3.6.4 基线作用原则

个体基线不是替代绝对阈值，而是作为修正层。

**最终原则：**
- **黄色判断**：既看绝对阈值，也看是否明显偏离个体常态。
- **红色、黑色判断**：以绝对阈值为主，个体基线为辅助说明层，不作为阻断条件。

### 3.7 第一阶段预警分级规则

#### 3.7.1 绿色
**同时满足：**
1. W_Ratio < 15%
2. W_Event_Rate < 15%
3. Consecutive_Days ≤ 1

**说明：** 整体异常占比较低，实时异常事件频率低，无连续多天异常状态，属于正常波动范围。

#### 3.7.2 黄色候选条件
**满足任一即进入黄色候选：**
1. 15% ≤ W_Ratio < 30%
2. Consecutive_Days = 2
3. Peak_Day_Flag = True
4. W_Ratio ≥ B_Ratio + 8个百分点
5. W_Event_Rate ≥ B_Event_Rate + 8个百分点

**说明：** 开始出现轻度但值得关注的波动。异常尚未达到中高风险水平，但已表现出持续、突发或明显偏离个人常态的趋势。

#### 3.7.3 红色候选条件
**满足任一即进入红色候选：**
1. 30% ≤ W_Ratio ≤ 50%
2. Consecutive_Days = 3 或 4
3. W_Event_Rate ≥ 33.3%
4. 两个连续周区间均为黄色确认，且趋势持续上升

**说明：** 异常状态已超出普通波动范围，呈现较明显的持续性和较高强度，需要重点关注和准备专业介入。

#### 3.7.4 黑色条件
**满足任一即直接进入黑色：**
1. W_Ratio > 50%
2. Consecutive_Days ≥ 5
3. W_Abnormal_2 > (W_Good + W_Neutral)

**说明：** 异常状态已在一周内成为主导状态，持续时间长、比例高，系统应直接进入高风险预警和专业转介准备。第三条等价于异常占比超过50%，保留这一写法便于理解“异常已经压过正常/正向状态”。

### 3.8 预警确认机制

**核心原则：** 系统不因第一次触发黄色条件就直接显示为黄色，而是先显示“趋势走向”。

#### 3.8.1 黄色确认机制
- 第一次触发黄色候选 → **绿色（黄色走向）**
  - 主等级仍为绿色
  - 旁边标注“黄色走向”
  - 存在轻微上升趋势，但尚未确认
- 第二次连续周区间再次触发黄色候选 → 正式升级为 **黄色**

#### 3.8.2 黄色趋势状态清除规则
如果第一次黄色候选后，下一周未再次触发黄色条件，则：
- 趋势标记自动清除
- 恢复为纯绿色

**绿色（黄色走向）** 只是观察态，不是永久状态。

#### 3.8.3 红色确认机制
- 第一次达到红色候选 → **黄色（红色走向）**
  - 保留黄色主体层级
  - 提示存在红色走向
  - 启动更强的第二阶段干预
- 下一周再次满足红色候选或明显上升 → 正式进入 **红色**

#### 3.8.4 黑色机制
黑色不采用双周确认。

**原因：** 黑色代表高风险，系统目标是预防黑色，不能因为过度确认而延迟干预。一旦满足黑色条件，直接进入黑色。

### 3.9 第一阶段输出内容

每周每个学生的输出应包括：
- W_Total（周总有效采样数）
- W_Abnormal_2（周异常样本数）
- W_Ratio（周异常占比）
- W_Event_1（周异常事件次数）
- W_Event_Rate（周异常事件归一化率）
- Consecutive_Days（连续异常天数）
- Peak_Day_Flag（单日异常峰值标记）
- 与个体基线比较结果
- 当前主等级
- 是否存在趋势标记

系统输出的不应只是一个颜色，而是一份**周级风险画像**。

## 四、第二阶段：家长端与老师端联动的轻干预与验证系统

### 4.1 阶段目标

第二阶段的目标是：
> 在不直接进入医疗化、咨询化路径的前提下，对第一阶段的预警结果进行家校联动验证和轻度干预

这是整个系统中最有价值的一层，在“什么都不做”和“直接看咨询师”之间建立了一个缓冲带。

### 4.2 第二阶段触发条件

第二阶段由以下状态触发：
- 绿色（黄色走向）
- 黄色
- 黄色（红色走向）
- 红色

**说明：**
- 纯绿色一般不进入第二阶段干预
- 黑色虽也会通知家长和老师，但同时直接进入第三阶段准备

### 4.3 家长端功能

家长端通过小程序或校园小程序实现，展示内容包括：
1. 当前等级
2. 趋势说明
3. 本周状态摘要
4. 建议动作
5. 是否需要与班主任协同
6. 是否需要进一步评估

#### 家长端展示原则

**1. 默认展示动作，不默认展示复杂数据**
家长默认看到的是：
- 风险提示
- 趋势说明
- 建议怎么做
而不是复杂统计表。

**2. 预警文案去诊断化**
统一表述为：
- 存在波动
- 存在趋势
- 建议观察
- 建议陪伴

而非：
- 孩子有问题
- 孩子异常
- 孩子心理状态差

### 4.4 老师端功能

老师端主要给：
- 班主任
- 学校心理老师
- 相关管理人员

**功能包括：**
1. 查看学生周状态
2. 接收预警通知
3. 记录在校观察结果
4. 提交反馈
5. 配合第二阶段干预判断

**分层授权：**
- 班主任：看趋势与建议
- 心理老师：可看更详细变化
- 管理层：看整体班级分布

### 4.5 第二阶段干预路径

#### 4.5.1 绿色（黄色走向）
**动作建议：**
1. 家长增加陪伴与观察
2. 注意孩子作息、活动、社交变化
3. 不进入正式心理咨询
4. 学校端可做轻量关注

**目标：** 验证是短期波动还是持续上升。

#### 4.5.2 黄色
**动作建议：**
1. 家长带孩子进行轻量活动释放
2. 可进入配备心理咨询师的运动场馆活动
3. 学校心理老师或班主任同步观察
4. 增强家校沟通

**目标：** 通过非医疗化场景观察孩子状态，判断是否缓解、维持或继续上升。

#### 4.5.3 黄色（红色走向）
**动作建议：**
1. 比黄色阶段更密集地观察
2. 家长与学校建立明确的周反馈机制
3. 优先安排系统内轻干预资源
4. 做下一周期重点追踪

**目标：** 验证是否真正进入红色。

#### 4.5.4 红色
**动作建议：**
1. 启动高强度第二阶段验证
2. 家长必须收到明确提示
3. 学校心理老师介入判断
4. 准备第三阶段专业资源转介

**目标：** 在尽量不滥用专业资源的前提下，不错过高风险学生。

### 4.6 第二阶段再评估机制

再评估结果分为三类：

#### 4.6.1 恢复
**满足任一即可：**
1. W_Ratio较上周下降 ≥ 10个百分点
2. Consecutive_Days中断
3. 重新回到绿色条件
4. W_Ratio接近B_Ratio范围

**结果：** 可降级或取消趋势标记。

#### 4.6.2 维持
**定义：** 指标波动不大，未明显下降，未明显恶化。

**结果：** 保持当前等级或趋势态，继续观察。

#### 4.6.3 恶化
**满足任一：**
1. W_Ratio继续上升
2. Consecutive_Days增加
3. 黄色进入红色候选
4. 红色进入黑色条件

**结果：** 升级处理。

## 五、第三阶段：专业介入与深度干预系统

### 5.1 阶段目标

第三阶段的目标是：
> 针对进入红色、黑色预警的学生，引入专业团队完成评估、干预和风险控制

### 5.2 第三阶段触发条件

**满足任一即可进入第三阶段建议通道：**
1. 正式红色
2. 黑色
3. 连续两周红色确认
4. 一周黑色且无下降
5. 异常趋势快速上升并持续

### 5.3 第三阶段资源来源

1. 心理咨询公司
2. 12355
3. 医院
4. 精神心理科或儿童心理相关资源
5. 自有心理健康团队

### 5.4 第三阶段执行逻辑

#### 1. 系统通知家长
明确说明：
- 当前属于高风险预警
- 建议进入专业评估
- 系统前期为预警系统，当前需要更专业判断

#### 2. 提供路径选择
家长可选择：
- 医院
- 合作机构
- 自有团队

#### 3. 记录转介结果
系统必须记录：
- 是否接受建议
- 选择了哪条路径
- 是否完成初次评估
- 后续结果反馈

### 5.5 强制升级线

**满足任一，系统必须建议进入第三阶段：**
1. 连续两周红色确认
2. 一周黑色且下一周无下降
3. 连续3天以上单日异常持续上升
4. 第二阶段干预无效且持续恶化

**目的：** 防止系统过度停留在观察期，防止错过真正需要专业介入的孩子。

## 六、三大核心风险与最终解决机制

### 6.1 风险一：误报导致信任损耗

**风险本质：** 不是系统会不会误报，而是误报后家长和老师还能不能继续信任系统。

**最终解决方式：**
1. 引入个体基线
2. 黄色、红色采用趋势确认机制
3. 黑色保留快速响应
4. 输出趋势而非绝对结论
5. 周级判断替代瞬时判断

### 6.2 风险二：家长焦虑和过度反应

**风险本质：** 不是家长会不会焦虑，而是系统有没有给出正确理解方式。

**最终解决方式：**
1. 家长端默认展示建议动作
2. 预警文案去诊断化
3. 周期性正常化提示
4. 趋势态先提醒，不直接贴标签
5. 红色之前尽量不进入正式咨询

### 6.3 风险三：升级边界不清导致流程失控

**风险本质：** 如果没有刚性边界，系统会出现该升不升、不该升过早升的情况。

**最终解决方式：**
1. 固定四级分层规则
2. 设置趋势确认机制
3. 第二阶段成为判断缓冲区
4. 设置强制升级线
5. 同时设置降级规则

## 七、系统最终运行逻辑总结

> **本系统通过问卷、实时异常事件和延迟高精度异常占比，按周对学生进行风险分层；对首次轻度异常先以趋势态提醒而非直接定级；再通过家长端与老师端进行轻干预和再评估；对持续上升或中高风险学生引入专业资源进行闭环干预**

## 八、最终版核心规则摘要

| 阶段 | 核心规则 |
|------|----------|
| **第一阶段** | 以周为区间；系统1负责异常事件次数，系统2负责异常占比；加入个体基线修正；黄色、红色采用趋势确认；黑色直接触发 |
| **第二阶段** | 家长端和老师端联动；轻干预、去医疗化、去标签化；干预后必须再评估；可恢复、维持或恶化 |
| **第三阶段** | 红色、黑色进入专业资源建议通道；形成转介闭环；设置强制升级线 |

## 九、最终结论

这套系统是具有以下特征的完整机制：

1. **有清晰定位**：只预警，不诊断
2. **有清晰阶段**：发现、验证、专业介入
3. **有清晰计算口径**：异常事件次数、异常占比、连续异常、峰值日、个体基线
4. **有清晰升级逻辑**：趋势态、确认态、正式态
5. **有清晰风险控制**：防误报、防家长焦虑、防流程失控
6. **有清晰闭环**：预警、干预、复核、转介、回流

### 核心优势
利用硬件和软件，**不是去判定有没有病**，而是：
> 在问题还没有被正式定义为问题之前，就建立起一个低侵入、可持续、可升级的预警与干预机制。
