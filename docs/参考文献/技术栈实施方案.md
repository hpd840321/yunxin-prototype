## 第一部分：项目概述

### 1.1 项目背景与目标

面向学校班级场景，构建一套**定时采集、异步分析**的人脸识别与情绪分析平台。系统通过定时拍摄班级全景照，在无实时性要求的时间窗口内（N+1天或N+0.5天）完成人脸识别和情绪分析，为教师提供班级情绪趋势看板、个体情绪追踪及数据脱敏推送能力。

### 1.2 核心业务流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     定时采集阶段（每日）                         │
│   ┌─────────┐  ┌─────────┐  ┌─────────┐                      │
│   │ 班级A   │  │ 班级B   │  │ 班级C   │  → 全景照批量存储    │
│   │ 全景照  │  │ 全景照  │  │ 全景照  │                      │
│   └─────────┘  └─────────┘  └─────────┘                      │
└───────────────────────────┬────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     异步分析阶段（夜间批量执行）                  │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │  全景照 → 人脸检测 → 人脸裁剪 → 特征提取 → 识别 → 情绪分析 │  │
│   │  (批量排队处理，充分利用NPU)                              │  │
│   └──────────────────────────────────────────────────────────┘  │
└───────────────────────────┬────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                     结果存储与推送                               │
│   ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│   │ SQLite存储   │→│ 教师看板查询 │→│ 脱敏推送后端 │       │
│   │ (历史/统计)  │  │ (部分脱敏)   │  │ (完全脱敏)   │       │
│   └──────────────┘  └──────────────┘  └──────────────┘       │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 建表SQL

```sql
-- 1. 班级表
CREATE TABLE class (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(50) NOT NULL,
    grade VARCHAR(20) NOT NULL,
    student_count INTEGER DEFAULT 0,
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_class_name ON class(name);

-- 2. 学生表
CREATE TABLE student (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    class_id INTEGER NOT NULL,
    name VARCHAR(50) NOT NULL,
    student_no VARCHAR(20) UNIQUE,
    face_feature BLOB,  -- 512维特征向量，约4KB
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (class_id) REFERENCES class(id)
);
CREATE INDEX idx_student_class ON student(class_id);
CREATE INDEX idx_student_no ON student(student_no);

-- 3. 分析记录主表
CREATE TABLE record (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    class_id INTEGER NOT NULL,
    capture_time DATETIME NOT NULL,
    photo_path VARCHAR(200),
    status TINYINT DEFAULT 0,  -- 0:待分析 1:分析中 2:已完成 3:失败
    total_faces INTEGER DEFAULT 0,
    recognized_count INTEGER DEFAULT 0,
    analyze_time DATETIME,
    error_msg TEXT,
    pushed TINYINT DEFAULT 0,   -- 0:未推送 1:已推送
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (class_id) REFERENCES class(id)
);
CREATE INDEX idx_record_class_time ON record(class_id, capture_time);
CREATE INDEX idx_record_status ON record(status);

-- 4. 明细记录表
CREATE TABLE record_detail (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    record_id INTEGER NOT NULL,
    student_id INTEGER NOT NULL,
    emotion_type VARCHAR(20),  -- happy/sad/angry/surprise/neutral
    emotion_score REAL,
    confidence REAL,
    face_location TEXT,        -- JSON: {x,y,w,h}
    crop_photo_path VARCHAR(200),
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (record_id) REFERENCES record(id),
    FOREIGN KEY (student_id) REFERENCES student(id)
);
CREATE INDEX idx_detail_record ON record_detail(record_id);
CREATE INDEX idx_detail_student ON record_detail(student_id);
CREATE INDEX idx_detail_emotion ON record_detail(emotion_type);

-- 5. 脱敏视图（查询时动态脱敏）
CREATE VIEW v_record_detail_masked AS
SELECT 
    rd.record_id,
    rd.emotion_type,
    rd.emotion_score,
    rd.confidence,
    CASE 
        WHEN length(s.name) = 2 THEN substr(s.name, 1, 1) || '*'
        WHEN length(s.name) >= 3 THEN substr(s.name, 1, 1) || '**'
        ELSE s.name
    END AS student_name_masked,
    CASE 
        WHEN length(s.student_no) >= 6 THEN 
            substr(s.student_no, 1, 4) || '****' || substr(s.student_no, -2)
        ELSE s.student_no
    END AS student_no_masked,
    rd.create_time
FROM record_detail rd
JOIN student s ON rd.student_id = s.id;
```

### 8.2 脱敏实现

```go
// pkg/mask/mask.go
package mask

func MaskName(name string) string {
    runes := []rune(name)
    if len(runes) <= 1 { return "*" }
    if len(runes) == 2 { return string(runes[0]) + "*" }
    return string(runes[0]) + strings.Repeat("*", len(runes)-1)
}

func MaskStudentNo(no string) string {
    if len(no) <= 6 { return "****" + no[len(no)-2:] }
    return no[:4] + "****" + no[len(no)-2:]
}

type Level int
const (
    LevelNone Level = iota
    LevelPartial
    LevelFull
)
```

### 8.4 HMAC签名防篡改

```go
func SignPayload(payload *PushPayload, secret string) string {
    data := fmt.Sprintf("%d|%s|%s|%s|%d",
        payload.RecordID, payload.ClassName,
        payload.CaptureTime.Format(time.RFC3339),
        payload.Stats.TopEmotion, payload.Stats.Total)
    h := hmac.New(sha256.New, []byte(secret))
    h.Write([]byte(data))
    return hex.EncodeToString(h.Sum(nil))
}

// internal/scheduler/scheduler.go
package scheduler

import "github.com/robfig/cron/v3"

func SetupScheduler() *cron.Cron {
    c := cron.New(cron.WithSeconds())
    
    // 每日14:00采集班级全景照
    c.AddFunc("0 0 14 * * *", func() {
        captureClassPhotos()
    })
    
    // 每日02:00执行批量分析
    c.AddFunc("0 0 2 * * *", func() {
        runBatchAnalysis()
    })
    
    // 分析完成后推送
    c.AddFunc("0 30 2 * * *", func() {
        pushResults()
    })
    
    return c
}
```

## 第十五部分：关键决策回顾

| 决策点 | 最终方案 | 理由 |
|--------|----------|------|
| 硬件平台 | RK3576（4GB/32GB） | 6TOPS NPU够用，成本最优 |
| 处理模式 | 定时采集+异步批量分析 | 无实时要求，错峰运行 |
| 数据库 | SQLite（WAL模式） | 轻量、零配置、支持SQL查询 |
| 模型选型 | 轻量版（Mobile系列） | 模型<20MB，推理速度快 |
| 活体检测 | 禁用 | 班级场景无需防攻击 |
| 前端UI | Vue3+Naive UI | 轻量，打包体积小 |
| 可视化 | ECharts按需加载 | 避免引入冗余图表库 |
| 数据脱敏 | 分级脱敏（部分/完全） | 平衡隐私与可用性 |
| 推送安全 | HMAC-SHA256签名 | 防篡改、防伪造 |
| 功耗管理 | 定时唤醒+休眠 | 降低能耗，延长寿命 |
